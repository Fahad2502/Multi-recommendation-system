# RecoHub API Documentation

Base URL: `http://localhost:8000`
Interactive docs: `http://localhost:8000/docs` (Swagger UI)

---

## Endpoints

### GET /health
```
GET /health
```
Response `200`:
```json
{ "status": "ok", "version": "2.0.0", "timestamp": "2026-07-04T11:00:00.000Z" }
```

### POST /recommend
Main endpoint. Returns movies, music, and books related to the input movie.

```
POST /recommend
Content-Type: application/json
```
Body:
```json
{ "movie": "Inception" }
```

Response `200`:

### GET /movie-details/{movie_id}
Returns full details for a single movie by its TMDB ID.

```
GET /movie-details/27205
```

Response `200`:
```json
{
  "title": "Inception",
  "overview": "A thief who steals corporate secrets...",
  "rating": 8.8,
  "release": "2010-07-16",
  "genres": ["Action", "Science Fiction", "Adventure"],
  "runtime": 148,
  "cast": [
    { "name": "Leonardo DiCaprio", "profile": "https://image.tmdb.org/t/p/w185/..." },
    { "name": "Joseph Gordon-Levitt", "profile": "https://image.tmdb.org/t/p/w185/..." },
    { "name": "Elliot Page", "profile": "https://image.tmdb.org/t/p/w185/..." }
  ],
  "trailer": "YoHD9XEInc0"
```

Error `404` — TMDB returned no results:
```json
{ "detail": "Movie not found: ..." }
```

## Caching

All `/recommend` and `/movie-details` responses are cached in memory for **1 hour**.
Cached responses include `"cached": true` in the `meta` field.

## Music sources

The `source` field in each music result indicates the provider:
- `"spotify"` — from Spotify API (higher quality metadata)
- `"itunes"` — from iTunes Search API (fallback, always available, includes 30s preview MP3)
